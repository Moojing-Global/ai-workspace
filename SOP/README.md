## How to Use SOPs

### For Specific Tasks
1. Identify the task you need to perform
2. Find the relevant SOP from the list above
3. Follow the step-by-step procedure
4. Use quality checklists to verify your work
5. Document your work in the [Tasks directory](../Tasks/) if significant

### For Creating New SOPs
When creating a new SOP document:

1. **Identify the need**: Is this a repeatable process that needs standardization?
2. **Choose clear naming**: Use lowercase with hyphens (e.g., `task-name-here.md`)
3. **Follow SOP structure**:
   - Overview/Purpose
   - When to use
   - Prerequisites
   - Step-by-step procedure
   - Quality checklist
   - Troubleshooting
   - Related documentation

4. **Add to this README**: Update the list above
5. **Update main README**: Add reference in [main documentation index](../README.md)
6. **Cross-reference**: Link from relevant context documents

## SOP Maintenance

### When to Create a New SOP
- A task is performed repeatedly
- Consistency is critical
- Multiple people need to perform the same task
- Best practices need to be documented
- Onboarding new team members

### When to Update an SOP
- Process changes or improves
- Common mistakes are identified
- New tools are adopted
- Feedback indicates confusion
- Quality issues arise

### Review Schedule
- **Monthly**: Review for accuracy and completeness
- **After process changes**: Update immediately
- **After feedback**: Incorporate improvements
- **During onboarding**: Test with new users
